# Когортний аналіз утримання користувачів | SQL, Google Sheets
SQL-проєкт із когортного аналізу: розрахунок Retention Rate, побудова матриці утримання та сегментація користувачів (Organic vs Promo).
**[Переглянути звіт у Google Sheets](https://docs.google.com/spreadsheets/d/116w1T9RHIm9tgl9aVnpdCMVDfNGT2Y4u9wtQbrppxvU/edit?gid=1745134236#gid=1745134236)**

### Мета проєкту:
Дослідити поведінку користувачів та розрахувати показник утримання (Retention Rate), щоб оцінити лояльність аудиторії та порівняти ефективність різних каналів залучення.

### Технологічний стек:
* **SQL:** (складні запити, CTE, віконні функції).
* **Data Cleaning:** очищення та нормалізація дат за допомогою регулярних виразів та операторів CASE.
* **Трансформація даних:** розрахунок month_offset для формування когорт.

### Ключові етапи реалізації:
1. **Підготовка та очищення:** Обробка сирих даних, нормалізація часових міток та підготовка таблиць cohort_users та cohort_events.
2. **Формування когорт:** Групування користувачів за місяцем їхньої першої активності.

### Аналіз в Google Sheets:
* **Побудова Retention-матриці:** Створення зведеної таблиці (Pivot Table) для відображення когорт по місяцях.
* **Умовне форматування (Heatmap):** Налаштування кольорової шкали для миттєвого визначення «критичних» зон відтоку користувачів.
* **Порівняльний аналіз сегментів:** Візуалізація різниці в утриманні між Organic та Promo каналами за допомогою графіків.
